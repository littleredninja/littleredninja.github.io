Let's get technical

and talk about blocks, procs and lambdas

Sunday, June 22, 2014

Blocks, procs and lambdas, besides being fun to say, are bits of code that perform an action. While these three are similar, they have a few differences.

Blocks

We'll start with blocks, which is a block of code that performs an action. They are usually written between curly braces, or between do and end. Several ruby methods take blocks, so if you've been studying Ruby for a little bit, blocks should look rather familiar to you.

books = ["The Great Gatsby", "A Tree Grows in Brooklyn", "Pride and Predjudice"]
books.each { |book| puts "#{book} is a great book!" }
 ==> "The Great Gatsby is a great book!"
 ==> "A Tree Grows in Brooklyn is a great book!"
 ==> "Pride and Prejudice is a great book!"

The code between the curly braces, or |book| puts "#{book} is a great book!", is a block! Blocks are not objects, and so they must be rewritten each time you need them. This gets a bit repetitive if you need to use a block more than once, but that's when we could use a proc.

Procs

Procs are essentially named blocks. Unlike blocks, procs are an object in Ruby and can be used more than once. This is helpful if you'll need to use your block more than once.

books = ["The Great Gatsby", "A Tree Grows in Brooklyn", "Pride and Predjudice"]
declare = Proc.new { |item| puts "#{item} is great!" }
books.each(&declare)
 ==> "The Great Gatsby is great!"
 ==> "A Tree Grows in Brooklyn is great!"
 ==> "Pride and Prejudice is great!"
movies = ["Ferris Bueller", "The Blues Brothers", "Chicago"]
movies.each(&declare)
 ==> "Ferris Bueller is great!"
 ==> "The Blues Brothers is great!"
 ==> "Chicago is great!"

To make a proc, you have to use Proc.new. To use a proc, make sure you write the name of the proc preceded by an ampersand & to show that it's a proc.

Lambdas

Lambdas are similar to procs, but with a few differences. They perform the same actions as procs, and are also objects in Ruby. You declare them by using the keyword lambda, and call them using the ampersand &.

books = ["The Great Gatsby", "A Tree Grows in Brooklyn", "Pride and Predjudice"]
declare = lambda { |item| puts "#{item} is great!" }
books.each(&declare)
 ==> "The Great Gatsby is great!"
 ==> "A Tree Grows in Brooklyn is great!"
 ==> "Pride and Prejudice is great!"

Lambdas differ from procs however, in that lambdas will check the number of arguments passed to it, while a proc does not. A proc will merely ignore extra arguments or return nil for missing arguments, while a lambda returns an argument error.

Also, lambdas act differently within methods than blocks. If you call a lambda within a method, the lambda will pass control back to a method after executing, while procs will return immediately.

def better_proc
  movies_win = Proc.new { return "Movies are better than books!" }
  movies_win.call
  return "Books are better than movies!"
 ==> "Movies are better than books!"
def better_lambda
  books_win = lambda { return "Movies are better than books!" }
  books_win.call
  return "Books are better than movies!"
 ==> "Books are better than movies!"
