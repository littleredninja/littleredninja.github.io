Let's get technical

and talk about regular expressions

Tuesday, July 1, 2014

Regular expressions, abbreviated to REGEX or REGEXP, is special text used to search for patterns. A simple example would be using a wildcard like *.txt to search for all text files, only REGEX is more powerful and can be used to search for more complex patterns.

While it often looks like a confusing string of characters, REGEX is actually made of simple codes to represent different patterns. They can be strung together and used to search for a wide variety of characters and even spaces.REGEX can help simplify code, and often several lines of code can be condensed to just one with a simple REGEX.

Getting started

REGEX expressions in Ruby begin and end with a / character. A very basic REGEX expression can search for a particular character. Simply place your character between the forward slashes and you'll have a REGEX!

/A/ for example, will search for a capital A.

/a/ for example, will search for a lower-case a.

Other searches

You can also search for variations, which can be useful when dealing with hard-to-predict user input.

/gr[ea]y/ for example, will search for either grey or gray.

A dot works much like a star in a wildcard search, and will match any character except a line break character. For example, /gr.y/ will also return grey or gray, but will also return gr%y, so it's best used sparingly.

Possibilities

Not only can you match specific characters, but you can match words, spaces, digits and more.

/\w/ will match any word.

/\d/ will match any single digit.

/\s/ will match any space.

There is a vast library of possibilities for matching. With so many options, the best way to find as many as possible is to play around and experiment with them!
